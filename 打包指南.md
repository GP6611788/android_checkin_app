# 智能查寝签到 APK 打包指南

## 📱 应用概述

**应用名称**：智能查寝签到  
**包名**：com.gsupl.checkin  
**版本**：1.0.0  
**目标API**：Android 7.0 - Android 14+ (APK 34)

## 🛠️ 打包环境要求

### 必要环境 (Linux / macOS)
- Ubuntu 20.04+ 或 macOS 10.15+
- Python 3.8+
- Java JDK 11+
- Android SDK 30+
- Android NDK 23c

### 推荐配置
- CPU: 4核+ (编译需较强性能)
- 内存: 8GB+
- 磁盘: 至少20GB可用空间

## 📦 完整打包步骤

### 第1步：准备Linux环境

#### 选项A: Windows WSL2 (推荐)
```bash
# 1. 启用WSL2
wsl --install Ubuntu

# 2. 启动Ubuntu
wsl

# 3. 更新系统
sudo apt update
sudo apt upgrade -y
```

#### 选项B: Linux服务器或虚拟机
确保网络连接稳定

### 第2步：安装依赖

```bash
# 1. 安装基础编译工具
sudo apt install -y \
    git zip unzip tar \
    python3 python3-pip python3-venv \
    openjdk-11-jdk \
    autoconf automake libtool pkg-config \
    zlib1g-dev libncurses5-dev libncursesw5-dev \
    libtinfo5 cmake libffi-dev libssl-dev \
    cython3 ninja-build \
    python3-setuptools

# 2. 安装Android SDK和NDK
# 下载Android Command Line Tools
wget https://dl.google.com/android/repository/commandlinetools-linux-9477386_latest.zip
mkdir -p ~/android-sdk/cmdline-tools
unzip commandlinetools-linux-9477386_latest.zip -d ~/android-sdk/cmdline-tools
mv ~/android-sdk/cmdline-tools/cmdline-tools ~/android-sdk/cmdline-tools/latest

# 设置环境变量
echo 'export ANDROID_HOME="$HOME/android-sdk"' >> ~/.bashrc
echo 'export ANDROID_SDK_ROOT="$ANDROID_HOME"' >> ~/.bashrc
echo 'export PATH="$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools"' >> ~/.bashrc
source ~/.bashrc

# 3. 安装SDK组件
yes | sdkmanager --licenses
sdkmanager "platform-tools" "platforms;android-34" "build-tools;34.0.0"
sdkmanager "ndk;23.2.8568313"

# 4. 安装Buildozer和依赖
pip3 install --upgrade pip
pip3 install wheel setuptools cython==0.29.37
pip3 install buildozer python-for-android

# 5. 创建虚拟环境 (可选)
python3 -m venv kivy_env
source kivy_env/bin/activate
```

### 第3步：准备项目文件

```bash
# 1. 创建项目目录
mkdir -p ~/android_project
cd ~/android_project

# 2. 将您的项目文件复制到这里
# 需要复制的文件：
# - main.py
# - checkin_core.py  
# - buildozer.spec
# - README.md
# - data/ (图标目录)

# 3. 检查文件结构
ls -la
```

### 第4步：添加应用图标

```bash
# 在data目录准备图标文件 (必需)
# icon.png - 应用图标 (512x512 PNG)
# presplash.png - 启动画面 (1024x768 PNG)

# 如果暂时没有图标，可以使用默认图标：
mkdir -p ~/android_project/data
wget -O ~/android_project/data/icon.png https://via.placeholder.com/512x512/1a237e/ffffff?text=Checkin
wget -O ~/android_project/data/presplash.png https://via.placeholder.com/1024x768/1a237e/ffffff?text=智能查寝签到
```

### 第5步：构建APK

```bash
# 1. 进入项目目录
cd ~/android_project

# 2. 初始化Buildozer (首次运行需要)
buildozer init

# 3. 检查配置文件
cat buildozer.spec | head -20

# 4. 清理之前的构建
buildozer android clean

# 5. 开始构建APK (第一次构建需要较长时间)
# 此步骤将下载依赖并编译，大约需要30-60分钟
buildozer android debug

# 6. 监视构建进度
tail -f .buildozer/android/platform/build-*/build-output.txt
```

### 第6步：获取生成的APK

```bash
# 构建成功后，APK文件位置：
# ~/android_project/.buildozer/android/platform/build-arm64-v8a/dists/checkin/checkin-1.0.0-debug.apk

# 复制到当前目录方便访问
find .buildozer -name "*.apk" -exec cp {} ./checkin_app.apk \;

# 检查文件大小
ls -lh *.apk
```

## 🔧 常见问题解决

### 问题1: Buildozer构建失败
```bash
# 清除缓存重试
rm -rf .buildozer
buildozer android clean
buildozer android debug
```

### 问题2: 内存不足
```bash
# 使用交换文件
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 问题3: 网络问题
```bash
# 设置代理 (如果需要)

```

## 🚀 生产发布准备

### 1. 生成签名密钥
```bash
keytool -genkey -v -keystore checkin.keystore \
  -alias checkin_key -keyalg RSA -keysize 2048 \
  -validity 10000
```

### 2. 构建发布版本
```bash
# 修改buildozer.spec
# android.release = 1

# 重新构建
buildozer android release
```

### 3. 对齐APK
```bash
zipalign -v 4 checkin_app.apk checkin_app_aligned.apk
```

### 4. 签名APK
```bash
apksigner sign --ks checkin.keystore \
  --ks-key-alias checkin_key \
  checkin_app_aligned.apk
```

## 📝 安全注意事项

### 敏感信息保护
1. **账号密码**：仅保存在内存中，应用关闭后清除
2. **网络传输**：使用AES加密传输
3. **本地存储**：不保存敏感数据到文件

### 权限最小化
应用仅请求必要权限：
- `INTERNET` - 网络连接
- `ACCESS_NETWORK_STATE` - 检查网络状态
- `ACCESS_WIFI_STATE` - Wi-Fi状态检查
- `WAKE_LOCK` - 防止锁屏
